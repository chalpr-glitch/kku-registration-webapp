## 📂 โครงสร้างโปรเจกต์
- `/src` - ซอร์สโค้ดแยกไฟล์สำหรับพัฒนาและทดสอบในเครื่องคอมพิวเตอร์
  - `Code.gs` - โค้ด Backend ของ Google Apps Script
  - `index.html` - โครงสร้างหน้าเว็บ
  - `style.css` - สไตล์การออกแบบ (สไตล์พรีเมียม, Responsive)
  - `app.js` - ระบบการค้นหารายชื่ออัจฉริยะ (Autocomplete) และแดชบอร์ด
- `/dist` - โค้ดที่รวมไฟล์กัน (Bundled) เพื่อนำไปใช้งานบน Google Apps Script
  - `Code.gs` - ไฟล์หลังบ้าน
  - `Index.html` - ไฟล์หน้าบ้าน (รวม CSS และ JS ไว้ในนี้แล้ว)

## 🚀 วิธีการนำไปติดตั้งบน Google Apps Script (GAS)

ทำตามขั้นตอนด้านล่างนี้เพื่อเชื่อมต่อระบบลงทะเบียนเข้ากับ Google Sheets ของคุณ:

### ขั้นตอนที่ 1: เปิด Script Editor บน Google Sheet
1. เปิดไฟล์ Google Sheet ของคุณ: [oas69](https://docs.google.com/spreadsheets/d/1knEtSncFxFQ39qCt-A6FC9dm5fzqBoDimD7Fr6fIe0A/edit)
2. ไปที่เมนู **ส่วนขยาย (Extensions)** -> **Apps Script**

### ขั้นตอนที่ 2: วางโค้ด Backend
1. ในหน้าต่าง Apps Script จะมีไฟล์ชื่อ `รหัส.gs` หรือ `Code.gs`
2. คัดลอกโค้ดทั้งหมดจากไฟล์ [dist/Code.gs](file:///Users/c_champ/.gemini/antigravity/scratch/kku-registration-webapp/dist/Code.gs) ไปวางแทนที่โค้ดเดิมทั้งหมดในโปรเจกต์
3. กดบันทึก (รูปแผ่นดิสก์)

### ขั้นตอนที่ 3: สร้างไฟล์หน้าบ้าน HTML
1. ในหน้าต่าง Apps Script คลิกปุ่ม **+ (เพิ่มไฟล์)** ข้างๆ คำว่า "ไฟล์"
2. เลือกประเภทไฟล์เป็น **HTML**
3. ตั้งชื่อไฟล์ว่า `Index` (ระบบจะใส่นามสกุล `.html` ให้โดยอัตโนมัติ)
4. คัดลอกโค้ดทั้งหมดจากไฟล์ [dist/Index.html](file:///Users/c_champ/.gemini/antigravity/scratch/kku-registration-webapp/dist/Index.html) ไปวางแทนที่โค้ดในไฟล์ `Index.html` ใหม่นี้ทั้งหมด
5. กดบันทึก

### ขั้นตอนที่ 4: เผยแพร่เว็บแอป (Deploy Web App)
1. คลิกปุ่ม **การทำให้ใช้งานได้ (Deploy)** ที่มุมขวาบน -> เลือก **การจัดการการทำให้ใช้งานได้ใหม่ (New deployment)**
2. คลิกรูปเฟืองข้างคำว่า "เลือกประเภท" -> เลือก **เว็บแอป (Web app)**
3. ตั้งค่ารายละเอียดดังนี้:
   - **คำอธิบาย:** การลงทะเบียนประจำเดือนมิถุนายน
   - **เรียกใช้ในฐานะ:** ตัวคุณเอง (อีเมลของคุณ)
   - **ผู้มีสิทธิ์เข้าถึง:** ทุกคน (Anyone) *เพื่อให้อุปกรณ์อื่นๆ เข้ามาลงทะเบียนได้โดยไม่ต้องเข้าสู่ระบบ Google*
4. คลิกปุ่ม **การทำให้ใช้งานได้ (Deploy)**
5. ระบบจะขอสิทธิ์การเข้าถึงข้อมูล (Authorize Access) ให้คลิก **ให้สิทธิ์การเข้าถึง (Review Permissions)** -> เลือกบัญชี Google ของคุณ -> คลิก **ขั้นสูง (Advanced)** -> คลิก **ไปที่... (Go to...)** -> กด **อนุญาต (Allow)**
6. คัดลอก **URL เว็บแอป** ที่ได้ เพื่อนำไปแชร์ให้ผู้เข้าร่วมงานหรือใช้เปิดเพื่อลงทะเบียนในวันงาน

## 🖥️ วิธีการเปิดพรีวิวทดสอบบนเครื่องตัวเอง (Local Development)

ระบบเขียนจำลองฐานข้อมูล Mock Database ไว้ ทำให้คุณสามารถทดสอบ UI และ Flow การทำงานทั้งหมดผ่านเบราว์เซอร์ของคุณก่อนขึ้นระบบจริงได้ทันที!

### 1. รันระบบ Local Dev Server
เปิด Terminal และพิมพ์คำสั่ง:
```bash
npm run dev
```
ระบบจะเปิดเบราว์เซอร์และแสดงหน้าจอลงทะเบียนให้ทดสอบทันที (พร้อมระบบ Hot Reload หากมีการแก้ไขไฟล์ใน `/src`)

### 2. ทดลองพิมพ์ค้นหารายชื่อ
ลองพิมพ์ชื่อของอาจารย์ในช่องค้นหา เช่น:
- *ชูชาติ*
- *ศุภสิทธิ์*
- *ประหยัด*
- *วรรณวิษา*
ระบบจะดึงรายชื่อจริงจากฐานข้อมูลมาให้ทำการยืนยันสเต็ปการลงทะเบียนจำลองอย่างราบรื่น

## 🎨 รายละเอียดฟีเจอร์การออกแบบระดับพรีเมียม
- **Glassmorphism Design:** พื้นหลังแบบฝ้าโปร่งแสงดูพรีเมียม สไตล์ Apple/modern web
- **Step Wizard Flow:** แยกการค้นหารายชื่อ -> ยืนยันตำแหน่ง/สังกัด -> การระบุความต้องการเพิ่มเติม เพื่อลดความซับซ้อนของผู้ใช้งาน
- **Real-time Admin Analytics:** แท็บแดชบอร์ดสรุปยอดรวม ยอดผู้ลงทะเบียน และผู้ที่ยังไม่ได้ลงทะเบียนแบบเรียลไทม์ พร้อมตารางกรองแผนกย่อยและการค้นหาอย่างคล่องตัว
- **Interactive Ticket:** เมื่อลงทะเบียนสำเร็จจะได้รับบัตรลงทะเบียน (Ticket) สไตล์สวยงามเพื่อแสดงหน้างาน
- **Mobile First:** รองรับการลงทะเบียนผ่านโทรศัพท์มือถือ แท็บเล็ต และคอมพิวเตอร์อย่างเป็นธรรมชาติ
